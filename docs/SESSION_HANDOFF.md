## Environment/process lessons (session 11 additions; session-9/10 lists still apply)

- **The backslash-u decode hazard also strikes VERBATIM report transcription.** Writing a reviewer's return via the Write tool re-emits any backslash-u escape the reviewer typed, and the tool-call JSON layer decodes it into a raw NUL inside the committed report (hit BOTH M2-T005 review reports; the G4 file even dodged git's binary detection because its NUL sat past the 8000-byte sniff window). Rule: byte-scan every transcribed report (python: bytes < 9 or 14–31) BEFORE committing; repair with the `bytes([92])+b'u0000'` construction, never by typing the escape; repair of already-merged evidence rides the next control PR with an honest note (precedent CP-0026).
- **SendMessage/agent-continuation does NOT exist in this environment.** A completed background agent cannot be resumed — relaunch a fresh agent pointed at the preserved report/design. Corollary: require producers to preserve their full design in the report BEFORE stopping (the M2-T006 stopped producer's report §7 made the A1 resume seamless).
- **Parallel dispatch works under auto mode:** two producers (and two reviewers) launched in one message ran concurrently with no permission rejection — the session-5 "launch sequentially" note is obsolete under posture v3.
- **Live-branch reconciliation pattern (extends the session-10 stale-PR pattern):** merge main into the second task branch → merge-commit tree MUST equal the `merge-tree --write-tree` dry-run tree (zero conflict content) → reviewed-scope diff old-head..new-head must be EMPTY (verdicts stay valid, no delta re-review) → full CI on combined tree → merge.
- **Packet STOP conditions work and are worth writing:** the explicit "if it needs client edits, stop and report" line in M2-T006's forbidden_paths halted the producer cleanly BEFORE implementation; the amendment went through its own control PR before resume.
- **Contract-version bumps are never purely backend (R-NEW-1):** the client pins a closed `SUPPORTED_CONTRACT_VERSIONS` runtime set, so every version publication is a coordinated atomic schema+backend+client change until the tooling task (generate the client array from the schema enum) lands. Scope future version-bump packets accordingly.
- `gh pr merge` without `--delete-branch` (used when a worktree holds the branch) leaves the REMOTE branch too — delete explicitly with `git push origin --delete <branch>` after removing the worktree, then verify via `ls-remote --heads`.
- Reviewer sandboxes could execute python/pytest/gh-read this session (both M2-T006 reviewers re-ran the full api suite themselves) — encourage direct re-execution in reviewer prompts; frontend reviewers remain code+CI-evidence-level (no browser/npm on the owner PC).
