## Environment/process lessons (session 9 additions)

- **CRITICAL: the Bash and PowerShell tools SHARE one persisted working directory.** A `cd` in a Bash call silently redirects later PowerShell commands (and vice versa). This caused ledger CLI runs against a WORKTREE's ledger copy ("Unknown task") and a `git pull` that fast-forwarded a task branch instead of main. Always `Set-Location`/`cd` to the repo root first, or use `git -C "<root>"` / absolute paths for every ledger/git/gh command.
- `gh pr create` run from the main checkout defaults head to main ("No commits between main and main") — always pass `--head <branch>`.
- PowerShell here-strings passed through the tool get mangled (parse errors mid-commit-message) — write commit messages/PR bodies to a temp file via the Bash tool heredoc and use `-F`/`--body-file`.
- `gh pr update-branch <n>` reruns CI cheaply after a base-branch fix lands (used to clear the S7 failure from PRs #20/#21); afterwards the local task worktree needs `git pull --no-rebase` before it can push (non-fast-forward).
- Producer agents SOMETIMES write directly to the assigned task worktree and sometimes get harness-isolated `agent-*` worktrees — always verify with `git -C <task-worktree> status --porcelain` before committing; transplant only if needed.
- CI web-e2e installs the api NON-editable (`pip install ./services/api`, no dev extras) — anything the service needs at runtime must be package data / runtime deps, never repo-relative paths or dev extras (the M2-T003 packaging lesson, now guarded by `contracts-schema-bundle`).
