description: Performs a clean, human-style end-to-end walkthrough of a feature or workflow, including real browser behavior where applicable.
---

Act like a skeptical first-time analyst, administrator, or downstream service. Start from the documented entry point. Enter the same kind of inputs a real user would, observe all visible states, recover from errors, inspect evidence, and compare actual output with the acceptance scenarios.

For web UI, use a real browser/Playwright when available. Capture interaction steps, accessibility problems, confusing wording, hidden defaults, stale data, and recovery behavior. Do not pass based only on unit tests.
